当前最早观察到的信号：

- `direct-only` 下：
  - `Stage A` 几乎完全可恢复
  - `Stage H` 几乎不可恢复
  - `Stage J / K` 处于中间
- `projection-enhanced` 下：
  - `Stage J / K` 的恢复率大幅上升，已逼近 `Stage A` baseline
  - `Stage H` 虽然上升，但仍明显低于部署线

这说明：

- 只看 `direct-only` 会低估部署线风险；
- 当前更强的 `VMA` 已足以揭示 `Stage J / K` 相对研究线的显著安全让步；
- `Gate 2` 表明：即便不看结构恢复，仅最小线性训练式反演器也足以高度恢复各类工件的 embedding 语义；
- `Gate 3` 表明：当前最小 hidden-state ISA 仅显示有限风险，而 attention-score 基线几乎无效；
- `Gate 4` 表明：频率攻击几乎完全由 token permutation 决定，因此 `Stage A/H/J/K` 在这条线上几乎无差异；
- `Gate 5` 表明：在可接受正确性区间内，调大 `alpha_e / alpha_h` 几乎不能显著降低核心攻击风险；
- `Gate 6` 表明：简单的敏感 token 定向加噪也不足以在可接受正确性区间内有效压低核心风险；
- 这意味着当前 6 个 Gate 的首轮安全分析已经形成闭环。

## 6. 已知约束

- 当前 Gate 1-6 结论主要对应 legacy conservative deployment line
- redesign `Stage H-K` 线尚未重跑 Gate 级安全评测
- `IA` 仍未实现
- `Stage J / Stage K` 更偏部署线，不能默认继承 `Stage H` 的安全强度
- 中间态攻击只计入部署时真实可能暴露的中间量，不计入本机 hook / recorder
