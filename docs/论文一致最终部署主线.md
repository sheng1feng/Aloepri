# Qwen 论文一致最终部署主线

## 1. 唯一目标

当前 Qwen 文档只保留一条主线：

> **论文一致的最终可交付部署线**

这条主线的最终目标是：

- 标准 Transformer 运行图
- 标准 `model.* / lm_head.*` 键布局
- 尽量保留论文要求的 attention / FFN / norm / embedding / head 参数扰动表达
- 形成最终可交付的 `Stage K` release

## 2. 当前状态总览

当前仓库已经不再把 legacy、buffered redesign、standard-visible bridge 视为三条并列进度线。

它们现在只承担三类证据职责：

- legacy：说明旧部署线为什么不够好
- buffered redesign：说明哪些安全性已经修回
- standard-visible bridge：说明标准键导出工程通路已经打通到哪里

当前唯一主线已经收口到：

- `Stage J` 唯一候选：`artifacts/stage_j_qwen_paper_consistent`
- `Stage K` 唯一 release 面：`artifacts/stage_k_release`
- 活跃 profile：`default` / `reference`

## 6. 当前安全性结果与剩余问题

以下清单只对应当前唯一活跃 Qwen 发布面：

- release surface：`artifacts/stage_k_release`
- active profiles：`default` / `reference`
- active security targets：`stage_k_default` / `stage_k_reference`

以下对象不再是当前主线复跑对象：

- buffered redesign
- standard-visible bridge
- 历史 `Stage J` deployment targets，例如 `stage_j_stable_reference`、`stage_j_tiny_a`

### 6.1 release-surface correctness

当前主线已经要求并承认的 correctness 证据只认 `Stage K` 自身路径：

- `outputs/stage_k_release/correctness/default.json`
- `outputs/stage_k_release/correctness/reference.json`
- `outputs/stage_k_release/correctness_summary.json`

当前可直接执行的最小检查入口是：

1. 用 `scripts/export_stage_k_release.py` 重新导出或确认 `artifacts/stage_k_release`
2. 用 `scripts/infer_stage_k_release.py` 分别对 `default` / `reference` 做 release-surface smoke；最小运行时至少需要补 `--prompt`

例如：

```bash
conda run --no-capture-output -n qwen-transformers python scripts/infer_stage_k_release.py \
  --release-dir artifacts/stage_k_release \
  --profile default \
  --prompt "请用一句话介绍你自己。"
```

这一步只能证明当前 release profile 仍然可运行、可见、可推理；最终 release-surface correctness 仍以这 3 份文件为准。

### 6.2 `VMA / IMA / ISA` 当前结果

当前主线只认 `stage_k_default` 与 `stage_k_reference` 两个安全目标。

当前两个 profile 来自同一个 `paper_consistent` 源工件，所以这轮结果在两个 profile 上一致。

#### `VMA`

- 执行入口：`scripts/security_qwen/run_vma.py`
- 当前结果文件：
  - `outputs/security_qwen/vma/stage_k_default.json`
  - `outputs/security_qwen/vma/stage_k_reference.json`
- 结果摘要：
  - `token_top1_recovery_rate = 0.0859375`
  - `risk_level = low`

#### `IMA`

- 执行入口：`scripts/security_qwen/run_ima.py`
- 历史最小基线：
  - 命令：`scripts/security_qwen/run_ima.py --mode minimal`
  - 结果文件：
    - `outputs/security_qwen/ima/stage_k_default.json`
    - `outputs/security_qwen/ima/stage_k_reference.json`
  - 结果摘要：
    - `token_top1_recovery_rate = 0.9765625`
    - `risk_level = high`
- 论文口径重测：
  - 命令：`scripts/security_qwen/run_ima.py --mode paper_like`
  - 结果文件：
    - `outputs/security_qwen/ima/stage_k_default.paper_like.json`
    - `outputs/security_qwen/ima/stage_k_reference.paper_like.json`
  - 结果摘要：
    - `token_top1_recovery_rate = 0.0`
    - `token_top10_recovery_rate = 0.0`
    - `risk_level = low`
- 当前主线只把 `.paper_like.json` 视为论文口径结论。
- 最小基线仍保留，是为了说明旧高风险信号来自“直接利用 `perm_vocab` 对齐混淆 embedding row 的 ridge 反演器”，它测的是 row-level learnability，不是论文中的小型神经 `IMA`。

#### `ISA`

- 执行入口：`scripts/security_qwen/run_isa.py`
- 结果必须显式记录 observable 类型
- 当前脚本支持：
  - `hidden_state`
  - `attention_score`
- `hidden_state` 结果文件：
  - `outputs/security_qwen/isa/stage_k_default.hidden_state.json`
  - `outputs/security_qwen/isa/stage_k_reference.hidden_state.json`
- `attention_score` 结果文件：
  - `outputs/security_qwen/isa/stage_k_default.attention_score.json`
  - `outputs/security_qwen/isa/stage_k_reference.attention_score.json`
- 结果摘要：
  - `hidden_state`: `intermediate_top1_recovery_rate = 0.0`，`risk_level = low`
  - `attention_score`: `intermediate_top1_recovery_rate = 0.0`，`risk_level = low`

聚合视图已经导出到：

- `outputs/security_qwen/summary/security_catalog.json`

### 6.3 完成判定

当前唯一 release 面已经补齐 correctness 与安全证据：

1. `artifacts/stage_k_release` 已重新导出或明确复核
2. `default` / `reference` 的 release-surface correctness 结果已落盘
3. `stage_k_default` / `stage_k_reference` 的 `VMA` 结果已落盘
4. `stage_k_default` / `stage_k_reference` 的 `IMA minimal` 与 `IMA paper_like` 结果已分开落盘
5. `stage_k_default` / `stage_k_reference` 的 `ISA` 结果已按 `hidden_state` 与 `attention_score` 两种 observable 落盘
6. 主线文档可以直接引用这些 `Stage K` release-surface 结果，而不是借历史 security 文档代指

但这不等于论文口径已经完全闭环。当前阻塞点已经从“`IMA` 结果缺失”转成“paper-like 攻击虽然已经回到低风险，但训练数据规模与部署参数还没有完全同态到论文配置”：

- `VMA`：低风险
- `IMA(paper_like)`：低风险
- `ISA(hidden_state / attention_score)`：低风险
- `IMA(minimal history baseline)`：高风险，但不再作为当前论文口径结论

## 7. 与原始论文的当前差异

当前 Qwen 主线已经收口到 `paper_consistent`，但还不能表述为“已经与论文完全等价”。

### 已经对齐的部分

- 已形成标准 Transformer 运行图与标准键布局
- 已形成标准 HF 工件与唯一 `Stage K` release 面
- 已经明确把 `bridge` 和 `buffered redesign` 降级为历史证据，而不是最终交付口径

### 仍未完全对齐的部分

- `export-visible completion` 只证明标准工件可导出、可见、可加载，不等于论文全部性质已经闭环证明
- 唯一 release 面上的 `VMA / IMA / ISA` 已经复跑，其中论文口径 `IMA` 已回到低风险；但这轮 `paper_like` 训练仍使用本地文档公共语料代理，而不是论文中更大规模的 public corpus
- 当前发布工件的混淆参数仍不是论文默认工作点：当前 `artifacts/stage_j_qwen_redesign/server/obfuscation_config.json` 记录的是 `alpha_e = 0.1`、`alpha_h = 0.05`，而论文推荐值是 `alpha_e = 1.0`、`alpha_h = 0.2`
- `default` / `reference` 当前仍是同源语义别名，后续如有需要可再分化

## 8. 下一步顺序

1. 如果要追求更严格的论文同态，在唯一 release 面上补一轮更接近论文规模的 public corpus `IMA` `paper_like` 复跑
2. 如果要追求更严格的参数同态，基于论文默认 `alpha_e = 1.0`、`alpha_h = 0.2` 重新物化并复跑 `correctness / VMA / IMA / ISA`
3. 根据后续 release 目标决定是否需要细化 `default` / `reference` 的独立工作点语义
