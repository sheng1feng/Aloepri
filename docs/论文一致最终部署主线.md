# Qwen 论文一致最终部署主线

## 1. 唯一目标

当前 Qwen 文档只保留一条主线：

> **论文一致的最终可交付部署线**

这条主线的最终目标是：

- 标准 Transformer 运行图
- 标准 `model.* / lm_head.*` 键布局
- 尽量保留论文要求的 attention / FFN / norm / embedding / head 参数扰动表达
- 形成最终可交付的 `Stage K` release

## 2. 当前状态总览

当前仓库已经不再把 legacy、buffered redesign、standard-visible bridge 视为三条并列进度线。

它们现在只承担三类证据职责：

- legacy：说明旧部署线为什么不够好
- buffered redesign：说明哪些安全性已经修回
- standard-visible bridge：说明标准键导出工程通路已经打通到哪里

当前唯一主线的阶段入口是：

## 3. 当前 `Stage H / I / J / K` 含义

### Stage H

定义论文允许保留、且仍可部署的参数表达集合。

### Stage I

验证这些表达是否仍在标准推理框架边界内。

### Stage J

把目标收束为论文一致的标准可交付部署物，而不是把 bridge 或 buffered redesign 当最终答案。

当前唯一候选部署物与证据出口为：

- `artifacts/stage_j_qwen_paper_consistent`
- `outputs/stage_j/paper_consistent/completion_summary.json`
- `outputs/stage_j/paper_consistent/correctness_regression.json`

### Stage K

把最终确认后的 `Stage J` 产物整理成唯一 release 面。

当前唯一 release 面与 profile 入口为：

- `artifacts/stage_k_release`
- `default`
- `reference`

当前 `Stage K` 直接继承 `paper_consistent` 的 correctness 证据，不再把任何 `bridge` 命名结果当作活跃 release correctness 语义：

- `outputs/stage_j/paper_consistent/correctness_regression.json`

## 4. 当前仍未完成的关键项

- 最终论文一致线上的 correctness 仍需在唯一 release 面同口径复跑
- 最终论文一致线上的 `VMA / IMA / ISA` 仍需在唯一 release 面同口径复跑
- `default` / `reference` 当前仍是同源语义别名，后续如有需要可再分化
