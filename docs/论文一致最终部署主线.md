# Qwen 论文一致最终部署主线

## 1. 唯一目标

当前 Qwen 文档只保留一条主线：

> **论文一致的最终可交付部署线**

这条主线的最终目标是：

- 标准 Transformer 运行图
- 标准 `model.* / lm_head.*` 键布局
- 尽量保留论文要求的 attention / FFN / norm / embedding / head 参数扰动表达
- 形成最终可交付的 `Stage K` release

## 2. 当前状态总览

当前仓库已经不再把 legacy、buffered redesign、standard-visible bridge 视为三条并列进度线。

它们现在只承担三类证据职责：

- legacy：说明旧部署线为什么不够好
- buffered redesign：说明哪些安全性已经修回
- standard-visible bridge：说明标准键导出工程通路已经打通到哪里

当前唯一主线已经收口到：

- `Stage J` 唯一候选：`artifacts/stage_j_qwen_paper_consistent`
- `Stage K` 唯一 release 面：`artifacts/stage_k_release`
- 活跃 profile：`default` / `reference`

## 6. 剩余复跑可执行清单

以下清单只对应当前唯一活跃 Qwen 发布面：

- release surface：`artifacts/stage_k_release`
- active profiles：`default` / `reference`
- active security targets：`stage_k_default` / `stage_k_reference`

以下对象不再是当前主线复跑对象：

- buffered redesign
- standard-visible bridge
- 历史 `Stage J` deployment targets，例如 `stage_j_stable_reference`、`stage_j_tiny_a`

### 6.1 release-surface correctness

当前主线已经要求并承认的 correctness 证据只认 `Stage K` 自身路径：

- `outputs/stage_k_release/correctness/default.json`
- `outputs/stage_k_release/correctness/reference.json`
- `outputs/stage_k_release/correctness_summary.json`

当前可直接执行的最小检查入口是：

1. 用 `scripts/export_stage_k_release.py` 重新导出或确认 `artifacts/stage_k_release`
2. 用 `scripts/infer_stage_k_release.py` 分别对 `default` / `reference` 做 release-surface smoke；最小运行时至少需要补 `--prompt`

例如：

```bash
conda run --no-capture-output -n qwen-transformers python scripts/infer_stage_k_release.py \
  --release-dir artifacts/stage_k_release \
  --profile default \
  --prompt "请用一句话介绍你自己。"
```

这一步只能证明当前 release profile 仍然可运行、可见、可推理；最终 release-surface correctness 仍以这 3 份文件为准。

### 6.2 `VMA / IMA / ISA` 复跑

当前主线只认 `stage_k_default` 与 `stage_k_reference` 两个安全目标。

#### `VMA`

- 执行入口：`scripts/security_qwen/run_vma.py`
- 最小运行时至少需要补 `--target stage_k_default` 或 `--target stage_k_reference`
- 预期结果文件：
  - `outputs/security_qwen/vma/stage_k_default.json`
  - `outputs/security_qwen/vma/stage_k_reference.json`

#### `IMA`

- 执行入口：`scripts/security_qwen/run_ima.py`
- 最小运行时至少需要补 `--target stage_k_default` 或 `--target stage_k_reference`
- 预期结果文件：
  - `outputs/security_qwen/ima/stage_k_default.json`
  - `outputs/security_qwen/ima/stage_k_reference.json`

#### `ISA`

- 执行入口：`scripts/security_qwen/run_isa.py`
- 最小运行时至少需要补 `--target` 与 `--observable-type`
- 结果必须显式记录 observable 类型
- 当前脚本支持：
  - `hidden_state`
  - `attention_score`
- 例如：
  - `outputs/security_qwen/isa/stage_k_default.hidden_state.json`
  - `outputs/security_qwen/isa/stage_k_reference.hidden_state.json`

如果需要统一比较视图，可以再用 comparison/export 脚本做汇总；但主线完成判定仍以每个 active target 的单独结果文件为准。

### 6.3 完成判定

只有同时满足下面几项，Qwen 当前唯一 release 面才算完成这轮主线复跑：

1. `artifacts/stage_k_release` 已重新导出或明确复核
2. `default` / `reference` 的 release-surface correctness 结果已落盘
3. `stage_k_default` / `stage_k_reference` 的 `VMA` 结果已落盘
4. `stage_k_default` / `stage_k_reference` 的 `IMA` 结果已落盘
5. `stage_k_default` / `stage_k_reference` 的 `ISA` 结果已按所选 observable 落盘
6. 主线文档可以直接引用这些 `Stage K` release-surface 结果，而不是借历史 security 文档代指

## 7. 与原始论文的当前差异

当前 Qwen 主线已经收口到 `paper_consistent`，但还不能表述为“已经与论文完全等价”。

### 已经对齐的部分

- 已形成标准 Transformer 运行图与标准键布局
- 已形成标准 HF 工件与唯一 `Stage K` release 面
- 已经明确把 `bridge` 和 `buffered redesign` 降级为历史证据，而不是最终交付口径

### 仍未完全对齐的部分

- `export-visible completion` 只证明标准工件可导出、可见、可加载，不等于论文全部性质已经闭环证明
- 论文主叙事涉及的 `VMA / IMA / ISA` 还没有在唯一 release 面上完成同口径复跑
- `default` / `reference` 当前仍是同源语义别名，后续如有需要可再分化

## 8. 下一步顺序

1. 在唯一 release 面上复跑 `VMA / IMA / ISA`
2. 根据复跑结果决定是否需要细化 `default` / `reference` 的独立工作点语义
