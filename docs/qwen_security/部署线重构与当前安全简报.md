# 部署线重构与当前安全简报

## 1. 这份简报回答什么

这份简报只回答两个问题：

1. 当前仓库的安全结论与论文主张相比差在哪里；
2. 这轮 `Stage H-K` 部署线重构，到底把哪些问题修掉了，哪些问题还没修掉。

## 2. 论文里的安全主张

论文的部署适配思路不是“把混淆尽量压成简单统一变换”，而是：

- 不改模型结构
- 在线推理图尽量保持标准 Transformer 形态
- 但尽量把复杂扰动吸收到 embedding / attention / FFN / norm / head 的参数表达中

在论文叙事下，AloePri 期望同时兼顾：

- 部署兼容性
- 功能正确性
- 对 `VMA / IMA / IA / ISA / TFMA / SDA` 等攻击的较强抵抗

## 3. 当前仓库在重构前的结论

重构前，仓库已经通过 Gate 1-6 得到一条很清楚的结论：

- 旧 `Stage J / K` 这条部署线是 **legacy conservative deployment line**
- 它虽然能标准化导出、能交付、能跑通
- 但它为了标准化交付，把很多本来应保留在参数表达里的复杂扰动压掉了

最直接的后果是：

- `VMA` 上旧 `Stage J` 接近 `Stage A` baseline
- `IMA` 上几乎所有线都高度可恢复
- `ISA hidden_state` 上旧 `Stage J` 也弱于 `Stage H`

所以旧部署线的问题不是“没导出成功”，而是：

> **为了标准部署而过度规整化，重新暴露了多组权重的共同结构。**

## 4. 当前重构后的状态

这轮 `Stage H-K` 重构之后，新的部署线已经和旧线拉开。

### 4.1 已经确认改善的部分

#### `VMA`

当前 focused comparison 里：

- 旧 `stage_j_stable_reference`
  - `top1 = 0.9765625`
- 新 `stage_j_redesign`
  - `top1 = 0.1015625`
- `stage_h_full_obfuscated`
  - `top1 = 0.1015625`

这说明：

> **新部署线在结构恢复面上已经明显贴近研究线，而不再接近旧 conservative line。**

#### `VMA source attribution`

对 redesign `Stage J`：

- `q / k / v` 单族几乎打不出恢复率
- `gate / up` 仍有一定信号，但显著弱于旧线

这说明旧线里“任一权重族都能高恢复”的问题已经明显缓解。

#### `ISA hidden_state`

当前 focused comparison 里：

- 旧 `stage_j_stable_reference`
  - `top1 = 0.0390625`
- 新 `stage_j_redesign`
  - `top1 = 0.0078125`
- `stage_h_full_obfuscated`
  - `top1 = 0.0078125`

这说明：

> **新部署线在 hidden-state 内部态保护上也已经回到接近 `Stage H` 的水平。**

### 4.2 还没有修掉的部分

#### `IMA`

当前 focused comparison 里：

- 旧 `stage_j_stable_reference`
  - `top1 = 0.9765625`
  - `cosine ≈ 0.99965`
- 新 `stage_j_redesign`
  - `top1 = 0.9765625`
  - `cosine ≈ 0.95074`

这说明：

- 结构侧确实改善了
- 但表示空间仍然高度可学习
- `IMA` 仍然是 redesign 线最主要的剩余问题

## 5. 这轮重构还没有完成的证明

虽然 redesign 线已经在 `VMA/ISA(hidden)` 上明显优于旧线，但当前还不能说它已经完全达到论文意义上的“可部署参数表达完成态”，原因是：

- `Stage J` export 仍然主要是 buffered stage-style safetensor 键
- 它还不是标准 `model.* / lm_head.*` 键布局
- 也就是说：
  - 已有 component-expression manifest
  - 但还没有完成“标准权重可见性证明”

这部分已经在：

- `outputs/stage_hk/redesign_expression_audit.json`
- `outputs/stage_i/phase2_feasibility.json`

中被明确记录。

## 6. 当前最准确的一句话

> 这轮 `Stage H-K` 重构已经证明：新部署线不是旧部署线的换名版本，它在 `VMA` 和 `ISA hidden_state` 上已经回到接近研究线 `Stage H` 的水平；但它还没有完成标准权重可见性证明，而且在 `IMA` 上仍然表现出很强的表示可学习性，因此距离论文理想中的部署适配安全性还差关键一步。
