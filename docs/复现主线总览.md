## 2. 整个复现流程摘要

当前仓库的完整复现流程可以压缩为 `Stage A -> K` 四段：

### Stage A-D：baseline 与早期恢复链路

- 建立明文基线、最小映射和早期恢复路径
- 验证基础 token-space / block-level 变换是否可闭环
- 给后续复杂结构恢复提供最小可工作的工程底座

### Stage E-G：复杂 attention、KeyMat 与结构表达恢复

- 推进复杂 attention 结构复现
- 逐步引入 KeyMat 与更完整的层内参数表达
- 把“能跑通”推进到“开始保留论文所需结构扰动”

### Stage H-I：可部署混淆表达与部署约束验证

- `Stage H` 定义哪些混淆表达在工程上仍可部署
- `Stage I` 检查这些表达是否还能落在标准推理框架边界内
- 这是从研究表达走向真实部署工件的分界点

### Stage J-K：标准可交付工件与 release 包装

- `Stage J` 把目标收束为标准可交付的混淆 HF 工件
- `Stage K` 把该工件收口为 release surface、profile、契约与入口

## 5. 当前部署线与原始论文的差异

当前仓库已经形成两条可交付的工程主线，但它们与原始论文理想目标并不完全等同。

### 已基本对齐的部分

- 都能导出标准 HF 形态的混淆工件
- 都形成了 client/server 输入映射与输出恢复闭环
- Qwen 与 Llama 都已经到达 `Stage K` 级别的 release surface

### 已可部署但仍非论文完全同态的部分

- Qwen 当前的 `paper_consistent` 线是“论文一致最终部署线”的工程近似收口，不等于论文中每个理论性质都已完成最终闭环证明
- Llama 当前主线已经在 correctness 与 release 层面成立，但仍沿用自身 `stable_reference / tiny_a` 语义，而不是 Qwen 的 `paper_consistent` 根口径

### 仍弱于论文理想或仍待补证的部分

- Qwen 还缺最终 release 面上的 correctness 同口径复跑
- Qwen 还缺最终 release 面上的 `VMA / IMA / ISA` 复跑
- Llama 还没有与论文主叙事完全一致的安全评测闭环
- 两条线都不能简单表述为“已经与论文完全等价”
